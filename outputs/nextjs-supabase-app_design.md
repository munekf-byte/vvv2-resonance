## 1. デザイン原則

### 1.1 基本理念

- **情報密度最優先**: スマホの限られた画面に最大限のデータを詰め込む
- **タップ最適化**: 全操作要素は `min-h-[34px]` 以上、`active:scale-95` フィードバック
- **色彩による即時認知**: 数値の意味を色で直感的に伝える（良好=緑, 危険=赤, 確定=金）
- **浮遊カードデザイン**: ウォームベージュ背景に白カードが浮き上がる構造で視認性確保
- **一貫性**: 同じ概念には同じ色・同じレイアウトを徹底

### 1.2 禁止事項（UIレギュレーション1）

| 禁止 | 理由 |
|------|------|
| `<select>` ドロップダウン | 横長・縦幅薄い入力は操作しづらい |
| `flex-1` で引き伸ばした薄いボタン群 | タップ領域が不均一 |
| 縦並びの複数 `<select>` スロット | 選択状態が視認しづらい |

### 1.3 必須事項

| 必須 | 仕様 |
|------|------|
| 全選択肢はボタングリッド形式 | `grid grid-cols-N gap-2` |
| ボタンの縦幅 | `py-3` 以上 |
| 選択状態の明示 | `boxShadow: "0 0 0 2px #1f2937"` |
| タップフィードバック | `active:scale-95` |
| トグルキャンセル | 再タップで選択解除（BITES種別/獲得等） |

## 4. レイアウト規則

### 4.1 セグメントコントロールタブ（v4.55〜）

全タブをiOS風の角丸セグメントコントロールに統一。

**ダッシュボード:**
```
┌── 稼働ログ ──┬── トータル数値分析 ──┐
│  (選択: 黒)   │  🔒 PRO限定 (金文字) │
└──────────────┴────────────────────┘
```
- 枠: `border: 2px solid #374151`, `rounded-lg`
- 選択: `#1f2937` 背景 / 非選択: `#f3f4f6` 背景
- PRO限定: 無課金時 `11px` ゴールド文字 / Pro時 `7px` 「PRO PLAN」

**Play画面:**
- 選択: `#dc2626`（赤）/ 非選択: `#1f2937`
- タブ間: `1px solid #4b5563` 区切り

### 4.2 浮遊カードデザイン

```
┌─── ヘッダー（sticky） ───┐
├─── ウォームベージュ背面 ──┤
│ ┌─ 白カード(黒枠+角丸+影) ─┐ │
│ │ メイン行 + ATサマリー     │ │
│ │ アコーディオン + ヤメ     │ │
│ └──────────────────────┘ │
│       space-y-1.5          │
│ ┌─ SESSION RESULT(独立) ─┐ │
│ │ 台差枚 | あなたの収支    │ │
│ └──────────────────────┘ │
└──────────────────────────┘
```

- **背面色**: `#e8e2d8`
- **カード**: `bg-white rounded-lg` + `border: 1px solid #000` + shadow
- **間隔**: `space-y-1.5`

### 4.3 通常時閲覧画面グリッド

```
grid-cols-[52px_1fr_1fr_1fr_46px_46px_46px_56px_26px]
```

| 列 | 内容 | 備考 |
|----|------|------|
| 52px | ✎ + 差枚数スタンプ(7px) | — |
| 1fr | 実G数 | **15px font-black 青(#1d4ed8)** + 「G」8px分離 |
| 1fr | ゾーン | ゾーン色 |
| 1fr | モード | 不明時: 上位3モード%表示 |
| 46px×3 | 契機/イベント/AT | — |
| 56px | 示唆 | 2行 |
| 26px | ▼ | アコーディオン |

## 6. AT記録画面

### 6.1 EDボナ選択時のセクション制御

`character === "EDボナ"` → 以下を非表示:
対決/直乗せ/不利益/BITES種別/BITES獲得/赫眼状態/終了画面示唆/トロフィー

代わりに表示:
- **赫眼カウンター**: ＋/−タップ（集計に反映、G数区分なし）
- **エンディングカード**: 常時展開（アコーディオン廃止）

### 6.2 枚数表示示唆

終了画面の直上に配置。3択トグル:
- `456 OVER` / `666 OVER` / `1000-7 OVER`
- `inferSetting` に自動反映

### 6.3 一時保存フィードバック

タップ → 緑(`#16a34a`) + `✓ 保存しました` → 1.2秒後復帰。全ダッシュボード共通。

## 8. 長押しヘルプ（LongPressHint）

- **発動**: 1.5秒長押し
- **テキスト選択防止**: `user-select: none` + `-webkit-touch-callout: none`
- **UI**: ダークヘッダーモーダル + OKボタン
- **対象**: FABボタン4種、タブ3種、SET行追加、ジャッジメント行追加

## 9. 差枚数計算エンジン

詳細: `docs/tg-medal-calculation.md`

```
Loss = totalNormalG × (50/31)
Gain = Σ(Base + SET数×40 + uenose) + czCount×32
Base: 通常AT=150 / エピボ=250 / ロンフリ=2000
```
