# COMMANDER LAB — クロス集計ダッシュボード アーキテクチャレポート

**作成日**: 2026-04-10
**対象バージョン**: v3.8
**目的**: 他プロジェクトでの再利用のための設計ドキュメント

---

## 1. 概要

COMMANDER LABは、全ユーザーの稼働データを**設定セグメント別**にクロス集計し、**Discord投稿用レポート**を生成する**管理者専用**ダッシュボードです。

**設計思想:**
- PC専用（情報密度・一覧性最優先）
- 15個の分析ブロック × 7個の設定セグメント = 巨大マトリクス
- 管理者（`is_admin === true`）のみアクセス可
- 論理削除データも含む全データが分析対象

## 4. 設定セグメント分類ロジック

### 4.1 セグメント定義（7列）

| セグメント | 意味 |
|-----------|------|
| 全体 | 全セッション（常に含まれる） |
| 推定低設定 | ユーザーが「推定低設定」と入力 |
| 推定4 | ユーザーが「推定4」と入力 |
| 推定456 | 設定4以上と推定されるセッション全て |
| 推定56 | 設定5以上と推定されるセッション全て |
| 推定6 | 設定6と推定されるセッション全て |
| 確定6 | 確定要素により設定6が濃厚なセッション |

### 4.2 優先順位ルール

```
確定要素あり → ユーザー推測を無視 → 確定ベースで強制分類
確定要素なし → ユーザー推測を採用
```

### 4.3 包含ルール（上位は下位に算入）

```
確定6 → {全体, 確定6, 推定6, 推定56, 推定456}
推定6 → {全体, 推定6, 推定56, 推定456}
推定56 → {全体, 推定56, 推定456}
推定4 → {全体, 推定4, 推定456}
推定低設定 → {全体, 推定低設定}
```

### 4.4 確定要素の検出ソース

| ソース | 検出対象 |
|--------|---------|
| CZ失敗終了画面 | `endingSuggestion` に「設定N濃厚」を含む |
| AT終了画面示唆 | SET行の `endingSuggestion` |
| トロフィー | 虹→確定6, 喰種柄→確定56, 金→確定456, 銀→確定3以上, 銅→確定2以上 |
| エンディングカード | `confirmed1-4` > 0 |

### 4.5 関数

```typescript
// セッションの確定要素を検出
function detectConfirmedSetting(session: PlaySession): string | null

// セッションが属するセグメントのSetを返す
export function classifySession(session: PlaySession): Set<SettingSegment>

// 全セッションをセグメント別に集計データ化
export function buildSegmentMap(sessions: PlaySession[]): Record<SettingSegment, SegmentData>
```

## 5. 15個の分析ブロック

### 5.1 ブロック一覧

| # | ブロック名 | データソース | 集計内容 |
|---|-----------|-------------|---------|
| 1 | 通常時 | `normalBlocks` | G数, CZ, エピボ, 直撃, AT初当たり（回数+確率） |
| 2 | CZ役別 | `czCounter` | 記録数, 成功率, 4役×(発生/当選/当選率) |
| 3 | 赫眼 | `kakugan[]` | 合計 + 各種別の出現数と割合 |
| 4 | 精神世界 | `shinsekai[]` | 合計 + 各種別の出現数と割合 |
| 5 | CZ失敗終了画面 | `endingSuggestion` | 各キャラの出現回数 |
| 6 | AT終了画面 | SET行の `endingSuggestion` | 各キャラの出現回数 |
| 7 | 有馬set | `atEntries` | 1/3/5set目の有馬出現率 |
| 8 | 招待状 | `invitation[]` | 設定示唆系の出現回数 |
| 9 | AT性能 | `normalBlocks` + `atEntries` | 初当たり, 引き戻し率, 裏AT突入率 |
| 10 | 対決成績 | SET行の `battles[]` | キャラ別勝率 |
| 11 | BITES | SET行の `bitesType` | テーブル別出現率 |
| 12 | ゾーン全体 | `normalBlocks` (all) | 加重按分込み9ゾーン分布 |
| 13 | ゾーン朝一 | `normalBlocks` (asaichi) | 朝一ブロックのみ |
| 14 | ゾーンAT後 | `normalBlocks` (afterAT) | AT後リセ頭のみ |
| 15 | 設定示唆サマリ | 全データ | セッション数, 総G数, トロフィー数 |

## 6. ゾーン集計の特殊ロジック

### 6.1 3つのフィルタモード

| モード | 判定条件 | 意味 |
|--------|---------|------|
| `all` | 全ブロック | 通常全体 |
| `asaichi` | uchidashi未設定 かつ index===0 | 朝一（設定変更後1発目） |
| `afterAT` | index > 0 かつ 前ブロック atWin===true | AT後リセ頭 |

### 6.2 加重按分マッピング

曖昧ゾーン（例: `300以内`）を確定ゾーン（50-600）に重み付け配分:

```typescript
"300以内": { "50": 0.05, "100": 0.05, "150": 0.1, "200": 0.3, "250": 0.1, "300": 0.4 }
```

200G/300G天井に比重を置いたゲーム仕様準拠の配分。

### 6.3 セッション独立性

```
❌ 全セッションのブロックを1配列に結合 → セッション境界で誤判定
✅ セッションごとに独立フィルタ → 結果をマージ
```

## 9. 堅牢性設計

### 9.1 JSONB安全アクセス

DBのJSONBカラムから取得したデータは型が不確実。以下のヘルパーで防護:

```typescript
function safeBlocks(session): NormalBlock[]   // normalBlocks を配列保証
function safeEntries(session): TGATEntry[]     // atEntries を配列保証
function sArr(v: unknown): string[]            // 配列保証
function sStr(v: unknown): string              // 文字列保証
```

### 9.2 セッション処理エラーのフォールバック

```typescript
for (const session of sessions) {
  try {
    const segs = classifySession(session);
    // ... 分類処理
  } catch (e) {
    // エラーセッションは「全体」に自動算入
    map["全体"].sessions.push(session);
  }
}
```

## 10. 他プロジェクトへの再利用ガイド

### 10.1 最小構成

1. **分析エンジン** (`analytics-admin.ts`): セグメント定義と分類ロジックをドメインに合わせて書き換え
2. **共通分析** (`analytics.ts`): フィルタモードと按分ロジックをドメインに合わせて定義
3. **API** (`route.ts`): 管理者認証 + 全データ取得
4. **ページ** (`page.tsx`): テーブル描画 + レポート生成

### 10.2 カスタマイズポイント

| 要素 | 変更方法 |
|------|---------|
| セグメント（列） | `SETTING_SEGMENTS` 配列を書き換え |
| 分析ブロック（行） | `ANALYSIS_BLOCKS` に追加し、`computeBlock` に `case` 追加 |
| 色彩 | `SEG_COLORS` オブジェクトを変更 |
| レポート形式 | `generateDiscordMarkdown` を書き換え |
| 認証 | `checkAdmin` の判定ロジックを変更 |

### 10.3 依存関係（外部ライブラリ）

- **Next.js** (App Router)
- **Supabase** (認証 + PostgreSQL)
- **React** (useState/useEffect)
- **追加ライブラリなし**（Chart.jsやD3等は不使用、純粋なHTMLテーブル）
