## 1. デザイン原則

### 1.1 基本理念

- **情報密度最優先**: スマホの限られた画面に最大限のデータを詰め込む
- **タップ最適化**: 全操作要素は `min-h-[34px]` 以上、`active:scale-95` フィードバック
- **色彩による即時認知**: 数値の意味を色で直感的に伝える（良好=緑, 危険=赤, 確定=金）
- **浮遊カードデザイン**: ウォームベージュ背景に白カードが浮き上がる構造で視認性確保
- **一貫性**: 同じ概念には同じ色・同じレイアウトを徹底

### 1.2 禁止事項（UIレギュレーション1）

| 禁止 | 理由 |
|------|------|
| `<select>` ドロップダウン | 横長・縦幅薄い入力は操作しづらい |
| `flex-1` で引き伸ばした薄いボタン群 | タップ領域が不均一 |
| 縦並びの複数 `<select>` スロット | 選択状態が視認しづらい |

### 1.3 必須事項

| 必須 | 仕様 |
|------|------|
| 全選択肢はボタングリッド形式 | `grid grid-cols-N gap-2` |
| ボタンの縦幅 | `py-3` 以上 |
| 選択状態の明示 | `boxShadow: "0 0 0 2px #1f2937"` |
| タップフィードバック | `active:scale-95` |
| トグルキャンセル | 再タップで選択解除（BITES種別/獲得等） |

## 4. レイアウト規則

### 4.1 浮遊カードデザイン（v4.26〜）

通常時一覧・AT記録一覧の共通デザイン。

```
┌─────────── ヘッダー（sticky） ───────────┐
├─────────── ウォームベージュ背面 ──────────┤
│  ┌── 白カード（rounded-lg + 1px黒枠 + 影）──┐  │
│  │ メイン行グリッド                          │  │
│  │ ATサマリーバー（AT当選時）                 │  │
│  │ アコーディオン（展開時）                   │  │
│  │ ヤメバナー（ヤメ時）                       │  │
│  └────────────────────────────────────────┘  │
│                 space-y-1.5                    │
│  ┌── 白カード ──────────────────────────────┐  │
│  │ ...次のブロック                            │  │
│  └────────────────────────────────────────┘  │
│                                                │
│  ┌── SESSION RESULT（独立浮遊カード）─────┐  │
│  │ 台差枚(概算) | あなたの収支              │  │
│  │    お疲れ様でした！                       │  │
│  └────────────────────────────────────────┘  │
└────────────────────────────────────────────┘
```

- **背面色**: `#e8e2d8`（ウォームベージュ。ブロック内クールグレーと差別化）
- **カード**: `bg-white rounded-lg` + `border: 1px solid #000` + `boxShadow: 0 1px 3px rgba(0,0,0,0.12)`
- **間隔**: `space-y-1.5`（6px）
- **ヘッダーとカードの幅一致**: データ行に水平パディングなし（列ズレ防止）

### 4.2 通常時閲覧画面（NormalBlockList）

### 4.3 ATサマリーバー

- **セットバック**: 26px余白 + 26px AT番号 = 52px（編集列右辺に揃え）
- **AT番号**: 深緑 `#14532d` 背景
- **外枠**: 上辺のみ `1.5px solid #14532d`（下辺・右辺は大外黒枠で代用）
- **上段**: set数 | BITES | 直のせ | CCG(条件付き) | 獲得 | 示唆
- **下段**: AT履歴ブロック（flex-wrap改行対応、高さ自動伸縮）

**合計獲得(概算)の計算:**
```
Base(150/250/2000) + SET数×40 + BITES合計 + 直乗せ合計 + CCG死神合計
```

| 表示箇所 | 配置 | デザイン |
|---------|------|---------|

| 通常時編集ダッシュボード | ヘッダー直下の帯 | 15px太字 白文字 緑/暗赤背景 |

### 4.5 SESSION RESULT

ヤメ記録がある場合、ブロック一覧の最下部に独立浮遊カードとして表示。

```
┌────────── SESSION RESULT ──────────┐
│  台差枚（概算）  │  あなたの収支    │
│    +1,200枚     │    +980枚       │
├──────────────────────────────────┤
│        お疲れ様でした！            │
└──────────────────────────────────┘
```

- **ヘッダー/フッター**: `#111827` 背景
- **左セル**: `computeFinalResult` の算出値
- **右セル**: 収支入力の `交換枚数 − 投資枚数`
- **色**: プラス=緑グラデ、マイナス=赤グラデ、未入力=`#374151`

### 4.6 ヤメバナー

- **位置**: ブロックカード内の最下部（角丸内に含まれる）

### 4.7 FABボタン群（通常時タブ下部）

**配置**: `fixed bottom-6`, 横並び中央寄せ, `gap-2`
**全ボタンに長押しヘルプ対応（1.5秒）**

## 5. AT記録画面

### 5.1 浮遊カードデザイン

通常時一覧と同一のウォームベージュ背面 + 黒枠角丸カード。

### 5.2 EDボナ選択時のセクション制御

`character === "EDボナ"` の場合、以下を**非表示**:
- 対決、直乗せ、不利益、BITES種別、BITES獲得、赫眼状態、終了画面示唆、トロフィー

代わりに表示:
- **赫眼カウンター**: ＋/−タップ（集計に反映、G数区分なし）
- **エンディングカード**: 常時展開（アコーディオン廃止）

### 5.3 枚数表示示唆

終了画面の直上に配置。3択トグル:
- `456 OVER` → 設定4以上濃厚（金背景 `#fde68a`）
- `666 OVER` → 設定6濃厚
- `1000-7 OVER` → 設定6濃厚

選択結果は `inferSetting` に自動反映。

### 5.4 一時保存フィードバック

タップ → ボタンが緑(`#16a34a`)に変化 + `✓ 保存しました` → 1.2秒後に復帰。
通常時/ATダッシュボード共通。

## 7. 長押しヘルプ（LongPressHint）

- **発動**: 1.5秒長押し
- **テキスト選択防止**: `user-select: none` + `-webkit-touch-callout: none`
- **UI**: ダークヘッダーモーダル + OKボタンで閉じる
- **対象**: FABボタン4種、タブ3種、SET行追加、ジャッジメント行追加

## 8. 差枚数計算エンジン

詳細は `docs/tg-medal-calculation.md` 参照。

- **Loss**: `totalNormalG × (50/31)` ≈ 1.613枚/G
- **Gain**: `Σ(Base + SET数×40 + uenose) + czCount×32`
- **Base**: 通常AT=150 / エピボ=250 / ロンフリ=2000
- **スタンプ**: イベント発生時の瞬間差枚数（AT獲得は次ブロックから反映）
- **打ち出し差枚数(samai)**: 全スタンプに加算

---

## 9. モード推定エンジン

詳細は `docs/tg-mode-estimation.md` 参照。

- **v2.0**: ベイズ尤度更新方式（旧confirm方式から移行）
- **前兆なし**: 尤度乗算（0.05〜0.95）で柔軟に更新
- **継承時Decay**: 0.95倍 + 均等配分で誤判定蓄積を防止
- **表示**: アコーディオン展開先にModeProbBar

## 12. デザイン判断の原則

### 12.1 色の選び方

| 情報の性質 | 色の方向性 | 例 |
|-----------|-----------|-----|
| ポジティブ | 緑系 | AT当選, プラス収支 |
| ネガティブ | 赤系 | マイナス収支, 同期エラー |
| 中立・情報 | 青系 | 総G数, 実G数テキスト |
| 特殊・レア | 紫/金系 | エンディングカード, 確定設定 |
| 金銭関連 | 緑系 | 収支, 獲得枚数 |
| 設定確定 | 金/黄系 | 枚数表示示唆, 確定バッジ |
| 終了・停止 | ネイビー | ヤメ |
