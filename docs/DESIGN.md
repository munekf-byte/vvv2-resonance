**作成日**: 2026-04-10

## 1. デザイン原則

### 1.1 基本理念

- **情報密度最優先**: スマホの限られた画面に最大限のデータを詰め込む
- **タップ最適化**: 全操作要素は `min-h-[34px]` 以上、`active:scale-95` フィードバック
- **色彩による即時認知**: 数値の意味を色で直感的に伝える（良好=緑, 危険=赤, 確定=金）
- **一貫性**: 同じ概念には同じ色・同じレイアウトを徹底

### 1.2 禁止事項（UIレギュレーション1）

| 禁止 | 理由 |
|------|------|
| `<select>` ドロップダウン | 横長・縦幅薄い入力は操作しづらい |
| `flex-1` で引き伸ばした薄いボタン群 | タップ領域が不均一 |
| 縦並びの複数 `<select>` スロット | 選択状態が視認しづらい |

### 1.3 必須事項

| 必須 | 仕様 |
|------|------|
| 全選択肢はボタングリッド形式 | `grid grid-cols-N gap-2` |
| ボタンの縦幅 | `py-3` 以上 |
| 選択状態の明示 | `boxShadow: "0 0 0 2px #1f2937"` |
| タップフィードバック | `active:scale-95` |

## 4. レイアウト規則

### 4.1 通常時閲覧画面（NormalBlockList）

**グリッド定義:**
```
grid-cols-[52px_1fr_1fr_1fr_46px_46px_46px_56px_26px]
```

| 列 | 幅 | 内容 | 配置 |
|----|----|------|------|
| 1 | 52px | 編集ボタン（✎） | 中央 |
| 2 | 1fr | 実G数 | 中央 |
| 3 | 1fr | ゾーン | 中央 |
| 4 | 1fr | モード | 中央 |
| 5 | 46px | 契機 | 中央 |
| 6 | 46px | イベント | 中央 |
| 7 | 46px | AT | 中央 |
| 8 | 56px | 示唆 | 中央(2行) |
| 9 | 26px | ▼アコーディオン | 中央 |

**行の高さ**: `min-h-[34px]`（v3.5.2で44px→34pxにコンパクト化）
**行罫線**: `border-b-2 border-gray-500`（太め=イベント区切りの明確化）
**列罫線**: `border-r border-gray-400`
**ヘッダー**: スティッキー固定（`sticky top-0 z-30`）

### 4.2 ATサマリーバー（通常時画面内インライン）

**レイアウト原則:**
- 上の通常行に**付随する項目**であることを示すため、左端を44pxセットバック
- AT番号セルのみ深緑塗り、他はライトモード（白背景+太枠）

**グリッド定義:**
```
gridTemplateColumns: "36px 38px 1fr 1fr 1fr auto"
```

| 列 | 内容 | 表示方式 |
|----|------|---------|
| AT番号 | `AT1` 等 | 深緑背景(`#14532d`), 白文字 |
| セット数 | `3set` | 数字14px太字 + "set"7px添え |
| BITES | `BITES 320枚` | ラベル6px + 数字11px **密着配置** |
| 直乗せ | `直乗せ 150枚` | 同上 |
| 合計獲得 | `合計獲得(概算) 620枚` | ラベル6px + 数字11px, 色`#14532d` |
| 示唆 | `奇数設定示唆` | 7px, hint表示（キャラ名ではない） |

**枚数表示の基本法則:**
- ラベルと数値は `mr-0.5` で密着（離れすぎない）
- 数値は `font-black`
- 単位「枚」は `text-[7px]` で数値の直後に添える

### 4.3 AT記録画面（ATBlockList）

**グリッド定義:**
```
grid-cols-[32px_32px_44px_30px_46px_34px_minmax(80px,1fr)_22px]
```

**ATヘッダー:**
```
gridTemplateColumns: "40px 42px 1fr 1fr 1fr auto"
```
- 枚数セル: ATSummaryCoinCellと同一デザイン
- ラベル色: 白(`#ffffff`)、合計は黄色(`#fde047`)
- 枚数に「枚」単位必須
- 合計は「合計獲得(概算)」= BITES + 直乗せ + ベースコイン(150 or 250)

### 4.4 ヤメバナー

- 位置: 当該ブロックの**直下**（時系列的に「この周期でやめた」を表現）
- 色: `#1e3a5f`（ネイビー）
- テキスト: `ヤメ · ○○G`、白文字、10px

### 4.5 FABボタン群（通常時タブ下部）

**配置**: `fixed bottom-6`, 横並び中央寄せ, `gap-2`

| ボタン | 色 | サイズ | 設計意図 |
|--------|-----|-------|---------|
| 打ち出し設定 | `#1e40af` (青) | `px-5 py-3.5`, `13px` | 状態設定系=青 |
| 収支入力 | `#059669` (緑) | `px-5 py-3.5`, `13px` | 金銭系=緑 |
| 推測設定 | `#92400e` (琥珀) | `px-5 py-3.5`, `13px` | 分析系=琥珀 |
| ＋周期追加 | `bg-v2-red` | **`px-6 py-4`, `15px`** | 最頻操作=**最大サイズ** |

**サイズの階層**: 周期追加は他の3つより `text-[15px]`/`px-6 py-4` で約15%大きい。最も頻繁に使うボタンを物理的に大きくすることで、操作効率を最大化。

## 5. 集計テーブル基本法則

### 5.1 テーブルグリッド構成

```
┌─────────────────────────────────────────────┐
│ カテゴリヘッダー（Cat）                        │
│ 背景: カテゴリ色(100%), テキスト: 白, 9px bold │
├─────────────────────────────────────────────┤
│ 列ヘッダー（THead）                           │
│ 背景: カテゴリ色(10%), テキスト: #6b7280, 7px │
│ 下罫線: 2px solid カテゴリ色                   │
│ 列間罫線: 1px solid #d1d5db                    │
├─────────────────────────────────────────────┤
│ データ行（TRow）                               │
│ 背景: 交互(白/#f7f7f7), 設定グレード色で上書き │
│ 下罫線: 1px solid #e5e7eb                      │
│ 列間罫線: 1px solid #e5e7eb                    │
│ 1列目: 8px左寄せ, 他列: 9px右寄せtabular-nums  │
└─────────────────────────────────────────────┘
```

### 5.2 絶対遵守ルール

1. **列ヘッダー**: カテゴリ同系色の薄い背景（`{color}18`）+ 太罫線
2. **縦罫線**: 全列間に `1px solid #e5e7eb`（セル状の見た目）
3. **横罫線**: 全行間に `1px solid #e5e7eb`
4. **セル中央揃え**: `alignItems: center`（上下中央）
5. **項目名左寄せ / 数値右寄せ**
6. **固定列幅**: 数値が確実に収まる幅（回数=44-52px, 確率=48-64px）
7. **`tabular-nums`**: 数値列で桁揃え
8. **`THead` + `TRow`** コンポーネントで統一実装

## 6. ダッシュボード設計

### 6.1 セッションカード

```
┌─────────────────────────────────┬──────┐
│ セッション名（13px bold, 2行まで）  │ 再開 │
│                                   │──────│
│ [総G数 1,200G] [AT初当たり 3回]   │ 削除 │
│ [収支 +500枚] [確定 4以上濃厚]     │      │
│ [推測 推定56]                     │      │
└─────────────────────────────────┴──────┘
```

**バッジレイアウト**: `grid-cols-2`（2列固定）
- 最後のバッジは `col-span-2`（全幅）
- データなしでも `—` 表示で位置維持
- 共通スタイル: `text-[9px] font-mono font-bold px-1.5 py-0.5 rounded`

### 6.2 セッション作成モーダル

**入力フィールド構成:**
1. 日付（`<input type="date">`、デフォルト今日）
2. ホール名（テキスト、max20文字）
3. 台番号（数値入力、max10文字）
4. **「上記の情報でセッション開始」**ボタン → 直接作成
5. プレビュー（`→ 04/08 マルハン新宿 [123]番台`）
6. セパレーター「または」
7. フリー入力（テキスト、max30文字）
8. **「この題名にする」**ボタン

**二重タップ防止**: `creating` フラグ + スピナーオーバーレイ

## 7. 特殊UIコンポーネント

### 7.1 画像付きモーダルピッカー（終了画面示唆・トロフィー）

**設計思想**: 経験が浅いユーザーがテキストだけでは判断できない → 実機画面と同じ画像で視覚的マッチング

**レイアウト:**
- タップでボトムシートモーダルが展開
- 各選択肢: サムネイル画像(56×56) + キャラ名 + 設定示唆テキスト
- 選択状態: `boxShadow: "0 0 0 2px #1f2937"`

### 7.2 エンディングカードアコーディオン

**トグルボタン:**
- 紫グラデーション（`linear-gradient(135deg, #7c3aed, #a855f7)`）
- 扇状カードデザイン: 5枚の画像を `-15°` ~ `+16°` で回転重ね
- 入力済み時: `○枚記録済` バッジ

**カード種別の画像表示:**
- 白/青/赤カード: 横長画像（`max-height: 72px`, `object-cover`）をカウンター行上部に
- 銅/確定カード: サムネイル画像（44×44）をカウンター行内に

### 7.3 CZ当選オーバーレイ

**Phase 1**: `after_hit.png`, `opacity: 0.85`, 即時表示
**Phase 2**: `after_hit_2.png`, `opacity: 0.85`, 4.5秒後に自動切替（サプライズ演出）
**背景画像**: `opacity: 0.08`（常時表示の薄い装飾）

## 11. デザイン判断の原則

### 11.1 色の選び方

| 情報の性質 | 色の方向性 | 例 |
|-----------|-----------|-----|
| ポジティブ | 緑系 | AT当選, プラス収支, 高設定 |
| ネガティブ | 赤系 | マイナス収支, 低設定, 同期エラー |
| 中立・情報 | 青系 | 総G数, 設定入力, 状態記録 |
| 特殊・レア | 紫/金系 | エンディングカード, 確定設定 |
| 金銭関連 | 緑系 | 収支, 獲得枚数 |
| 警告・注意 | 黄/橙系 | 未同期, 分析関連 |
| 終了・停止 | ネイビー | ヤメ |

### 11.2 サイズの決め方

| 使用頻度 | サイズ | 例 |
|---------|-------|-----|
| 最頻 | 最大 | 周期追加ボタン |
| 高頻度 | 標準 | 打ち出し/収支/推測 |
| 低頻度 | 小 | 画像で保存, X共有 |
| 表示のみ | 極小 | ラベル, バッジ |

### 11.3 罫線の使い分け

| 罫線 | 用途 |
|------|------|
| `2px solid` + カテゴリ色 | セクション区切り, ヘッダー下 |
| `border-b-2 border-gray-500` | イベント行区切り（通常時） |
| `1px solid #e5e7eb` | テーブル内部（行・列） |
| `1px solid #d1d5db` | 列ヘッダー内部 |
| `1.5px solid #14532d` | ATサマリーバー外枠 |
